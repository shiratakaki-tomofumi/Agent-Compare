## 画面パターン（代表例）

画面は以下の4パターンの組み合わせで構成される。各画面は該当パターンに従い、固有のフィールドは画面一覧のモデル定義（`data-model.md`）を参照。

### パターンA: 一覧画面（顧客一覧、商談一覧、案件一覧、経費一覧、従業員一覧）

構成要素:
- 検索テキストフィールド + ステータスフィルタ（ドロップダウン）
- [+新規] ボタン（権限に応じて表示）
- データテーブル（各モデルの主要フィールドをカラム表示）
- ページネーション（1ページ10件）
- 行クリックで詳細画面に遷移

### パターンB: 詳細画面（顧客詳細、商談詳細、案件詳細、従業員詳細）

構成要素:
- 基本情報の表示（モデルの全フィールド）
- [編集] [削除] ボタン（権限に応じて表示）
- 関連データの一覧表示（例: 顧客詳細→関連商談一覧、案件詳細→タスク一覧）

### パターンC: 作成/編集フォーム

構成要素:
- 各フィールドの入力フォーム（必須項目に `*` マーク）
- バリデーション（必須チェック、形式チェック）
- [キャンセル] [保存] ボタン
- 保存成功時: トースト通知 + 一覧に遷移

### パターンD: 特殊画面

| 画面 | 構成 |
|------|------|
| ログイン | メール + パスワード入力、エラー表示、成功時ダッシュボードへリダイレクト |
| ダッシュボード | 4つのKPIカード（営業/案件/財務/HR）横並び + アクティビティフィード（直近5件） |
| 経費承認 | 承認待ち一覧 + 各行に[承認][却下]ボタン + コメント入力ダイアログ |
| 収支サマリー | 月次切替 + 収入/支出/差額カード + 部門別予算vs実績テーブル + カテゴリ別内訳 |
| 部署管理 | 部署一覧テーブル（部署名、所属人数）+ インライン編集/削除 |

## 画面固有のフィールド・仕様

### ダッシュボードKPIカード

| モジュール | 表示KPI |
|-----------|---------|
| 営業 | 当月売上、商談件数、成約率 |
| 案件 | 進行中案件数、完了タスク数、遅延案件数 |
| 財務 | 当月支出、承認待ち件数 |
| 人事 | 従業員数、当月入社者数 |

### 案件一覧の固有要素
- 進捗率をプログレスバーで表示（完了タスク数 / 全タスク数）

### 案件詳細の固有要素
- タスク一覧（タスク名、担当者、期限、優先度、ステータス）+ [+タスク追加]ボタン

### 商談のステータス遷移
リード → 提案 → 交渉 → 成約/失注

### 商談固有フィールド
商談名、顧客（セレクト）、金額、確度（%）、ステータス、担当者、メモ

```
                          ├→ 案件 → 案件一覧 ⇄ 案件詳細（タスク管理含む）⇄ 案件編集
                          ├→ 財務 → 経費一覧 ⇄ 経費作成/編集

                          └→ 人事 → 従業員一覧 ⇄ 従業員詳細 ⇄ 従業員編集
                                  → 部署管理
```
