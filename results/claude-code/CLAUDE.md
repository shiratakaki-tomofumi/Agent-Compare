# BizBoard 実装指示

## 依頼者について

- 合理主義者。不要な確認は不要。判断できることは自律的に最善の形で進めること
- 実装後の検証を自分では行わない。**成果物がそのまま動くこと**を期待している
- 品質最重視。「とりあえず動く」は不可。セキュリティ・保守性・パフォーマンスの水準を満たすこと
- 中途半端より完全。全機能が半端より、実装した機能が確実に動作する方を評価する

これにより求められること:
1. 自己検証の徹底（ビルドエラー・型エラー・実行時クラッシュがないことを自分で確認）
2. 動かないコードを納品しない（未完成機能はスコープから外す）

## 指示

`design/` 配下の設計書に基づき、業務管理ダッシュボードアプリ「BizBoard」を実装してください。

### 参照する設計書

| ファイル | 内容 |
|---------|------|
| `design/requirements.md` | 機能要件（必ず全項目を確認） |
| `design/screen-specs.md` | 画面一覧・レイアウトパターン・画面遷移 |
| `design/data-model.md` | テーブル定義・リレーション |
| `design/tech-stack.md` | 技術スタック・制約事項（必ず遵守） |
| `design/quality-guidelines.md` | **品質基準（必ず遵守）** — 操作品質・セキュリティ・パフォーマンス・保守性 |

### 成果物

以下を含む、**そのまま動作する**プロジェクトを生成すること:

1. 全ソースコード
2. `README.md`（セットアップ手順: DB作成→マイグレーション→シード→起動）
3. `.env.example`
4. `prisma/schema.prisma`
5. `prisma/seed.ts`（デモ用データ）

### 補足

- 基本設計のみ提供。詳細設計は自身で判断すること
- 想定ファイル規模: 20〜30ファイル
